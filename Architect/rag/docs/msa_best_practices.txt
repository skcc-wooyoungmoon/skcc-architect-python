# 마이크로서비스 아키텍처(MSA) 모범 사례

1.  단일 책임 원칙 (Single Responsibility Principle): 각 서비스는 하나의 비즈니스 기능에만 집중해야 합니다.
2.  분산된 데이터 관리 (Decentralized Data Management): 각 마이크로서비스는 자체 데이터베이스를 소유하고 관리해야 합니다. 다른 서비스의 데이터베이스에 직접 접근해서는 안 됩니다.
3.  API 게이트웨이 (API Gateway): 클라이언트의 요청을 적절한 마이크로서비스로 라우팅하는 단일 진입점을 제공합니다. 인증, 로깅, 캐싱 등의 공통 기능을 처리할 수 있습니다.
4.  장애 격리 (Fault Isolation): 하나의 서비스 장애가 전체 시스템의 장애로 이어지지 않도록 설계해야 합니다. 서킷 브레이커 패턴이 유용합니다.
5.  CI/CD 자동화 (Automation): 각 서비스를 독립적으로 빌드, 테스트, 배포할 수 있는 완전 자동화된 CI/CD 파이프라인을 구축해야 합니다.
