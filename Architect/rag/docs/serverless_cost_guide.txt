# 서버리스 비용 최적화 가이드

1.  메모리 최적화: AWS Lambda와 같은 FaaS는 할당된 메모리 양에 따라 CPU 성능과 비용이 결정됩니다. 함수 실행에 필요한 최소한의 메모리를 할당하여 비용을 절감할 수 있습니다. PowerTuning과 같은 도구를 사용하여 최적의 메모리 크기를 찾으세요.
2.  콜드 스타트 최소화: 프로비저닝된 동시성(Provisioned Concurrency)을 사용하여 자주 호출되는 함수의 콜드 스타트를 방지할 수 있습니다. 이는 추가 비용이 발생하지만, 일관된 성능을 보장합니다.
3.  효율적인 코드 작성: 함수 실행 시간이 길어질수록 비용이 증가합니다. 외부 API 호출 시간을 줄이고, 불필요한 라이브러리를 제거하여 코드 실행 시간을 단축하세요.
4.  적절한 서비스 선택: 모든 워크로드에 Lambda가 최선은 아닙니다. 장시간 실행되는 작업에는 AWS Fargate나 EC2가 더 저렴할 수 있습니다.
5.  API Gateway 비용: API Gateway는 요청 수와 데이터 전송량에 따라 과금됩니다. 캐싱을 활성화하여 반복적인 요청에 대한 백엔드 호출을 줄이면 비용을 절감할 수 있습니다.
